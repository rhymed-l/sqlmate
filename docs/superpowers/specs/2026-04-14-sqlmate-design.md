# SQLMate 设计文档

> 创建时间：2026-04-14 | 状态：已确认，待实现

## 概述

SQLMate 是一个跨平台桌面工具，面向开发者和 DBA，提供 SQL 批量处理能力：合并、拆分、分割大文件、格式化美化。本地运行，无需联网，数据不离开用户机器。

---

## 技术栈

| 层级 | 选型 | 说明 |
|------|------|------|
| 桌面框架 | Tauri 2.0 | Rust 壳 + 系统 WebView，打包体积 ~10MB |
| 前端框架 | React 19 + TypeScript | 主力语言，生态丰富 |
| UI 组件 | shadcn/ui + Tailwind CSS | 高质感可定制，不额外增加体积 |
| SQL 处理 | TypeScript（前端层） | 合并/拆分/格式化均为字符串处理，无需 Rust |
| SQL 格式化库 | sql-formatter | 纯 JS，支持多方言，无需服务端 |
| 文件 I/O | Tauri FS API（Rust） | 读大文件、写多文件、选文件夹对话框 |
| 主题切换 | Tailwind `dark:` + CSS 变量 | 跟随系统 + 手动切换 |
| 打包 | `tauri build` | 独立 exe/dmg/AppImage，无需安装运行时 |

**选型关键理由：**
- 排除 Electron：打包体积 150MB+（内嵌 Chromium）
- 排除 Go + Wails：体积 50MB+，是 Tauri 的 5 倍
- 排除 Flutter：Windows 开发环境需 ~10GB（VS2022 C++ 工作负载），且 Dart 不熟悉
- Tauri：系统 WebView + Rust 壳，体积 ~10MB，前端用 TypeScript 主力开发

## UI 设计

### 视觉风格
- **主题**：深色 / 浅色双模式，跟随系统，支持手动切换
- **强调色**：紫蓝渐变（`#7c8cf8` → `#a78bfa`）
- **风格参考**：开发工具质感，类似 VS Code / GitHub Copilot

### 布局
- **整体**：左侧图标导航 + 右侧内容区
- **导航栏**：窄边栏，图标 + 文字标签，4 个功能模块
- **内容区**：步骤流式交互

### 步骤流式交互（所有功能页统一模式）
```
① 输入  →  ② 配置参数  →  ③ 查看结果
```
每个步骤用编号圆圈引导，清晰分区，配置和输入分离，避免界面拥挤。

```
sqlmate/

│   ├── components/

│   │   ├── StepFlow.tsx        # 步骤流式容器
│   │   ├── SqlEditor.tsx       # 输入区（粘贴 + 拖拽）
│   │   └── ResultPanel.tsx     # 输出区（复制 / 保存文件）
│   ├── pages/
│   │   ├── Merge.tsx           # SQL 合并页
│   │   ├── Split.tsx           # SQL 拆分页
│   │   ├── Segment.tsx         # SQL 分割页
│   │   └── Format.tsx          # SQL 格式化页
│   └── lib/sql/
│       ├── merge.ts            # 合并逻辑
│       ├── split.ts            # 拆分逻辑
│       ├── segment.ts          # 分割逻辑
│       └── format.ts           # 格式化（封装 sql-formatter）
│
└── src-tauri/                  # Rust / Tauri 层
    └── src/
        ├── main.rs             # 注册 Tauri 命令
        └── commands/
            └── file.rs         # 读文件、写文件、选文件夹对话框
```

**数据流：**
```
用户拖入文件
  → Tauri FS API 读取内容 → 传给前端 TypeScript
  → lib/sql/*.ts 处理
  → 结果显示在 ResultPanel
  → 用户点"保存" → 调用 Tauri 写文件命令
```

> SQL 处理全在 TypeScript 层，Rust 只做文件系统操作。逻辑易测试，Rust 部分极简。

## 功能规格

### 1. SQL 合并（Merge）

**目标**：将多条独立的 `INSERT INTO` 语句合并为批量插入语句。

**交互流程：**
- ① 输入：粘贴文本 或 拖拽/选择 `.sql` / `.txt` 文件
- ② 配置：每批条数（默认 1000，数字输入框）
- ③ 结果：合并后的批量 INSERT，支持复制到剪贴板 或 保存为 `.sql` 文件

**处理规则：**
- 解析每条 INSERT 的表名 + VALUES，按表名分组
- 同一表的 VALUES 合并，超过批次数则新起一条 INSERT
- 不同表的语句保持独立，不跨表合并
- 结果区显示"共处理 N 张表，生成 M 条语句"

---

### 2. SQL 拆分（Split）

**目标**：将批量 INSERT 语句拆回单条语句，与合并互为逆操作。

**交互流程：**
- ① 输入：粘贴文本 或 拖拽/选择文件
- ② 配置：无额外参数
- ③ 结果：每个 VALUES 对应一条独立 INSERT，支持复制 或 保存

**处理规则：**
- 识别 `VALUES (...),...,(...)` 模式，按括号组拆分
- 每条输出语句保留原始表名和字段列表

---

### 3. SQL 分割（Segment）

**目标**：将大 SQL 文件拆分为多个小文件，便于分批执行。

**交互流程：**
- ① 输入：拖拽/选择大 SQL 文件
- ② 配置：选择模式
  - 按条数：每文件 X 条语句（默认 10000）
  - 按大小：每文件不超过 X MB（默认 10MB）
- ③ 结果：选择输出文件夹 → 写入 `output_001.sql`、`output_002.sql`…（强制保存，无复制选项）

**处理规则：**
- 大文件使用 Tauri 流式读取，避免一次性加载进内存
- 输出文件名格式：`output_001.sql`，序号补零至三位

---

### 4. SQL 格式化（Format）

**目标**：美化 SQL 的缩进与结构，支持多方言。

**交互流程：**
- ① 输入：粘贴文本 或 拖拽/选择文件
- ② 配置：
  - 方言下拉：MySQL / PostgreSQL / SQL Server / Oracle / SQLite / BigQuery
  - 缩进：2 空格 / 4 空格
- ③ 结果：格式化后的 SQL，支持复制 或 保存

---

## 输入 / 输出规范

| 功能 | 输入方式 | 输出方式 |
|------|----------|----------|
| 合并 | 粘贴 / 拖拽文件 | 复制到剪贴板 + 保存为文件 |
| 拆分 | 粘贴 / 拖拽文件 | 复制到剪贴板 + 保存为文件 |
| 分割 | 拖拽文件 | 强制保存到文件夹（多文件） |
| 格式化 | 粘贴 / 拖拽文件 | 复制到剪贴板 + 保存为文件 |

支持文件类型：`.sql`、`.txt`

---

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 输入内容不是合法 SQL | 步骤 ③ 显示错误提示，不崩溃，允许重新编辑 |
| 拖入文件类型不支持 | 提示"仅支持 .sql / .txt 文件" |
| 合并时发现多张表混合 | 按表名自动分组，结果区显示"共处理 N 张表" |
| 分割时文件过大（1GB+） | Tauri 流式读取，避免一次性加载进内存 |
| 保存文件时路径无写权限 | 弹出系统错误提示，不静默失败 |
| 格式化遇到不支持的语法 | 降级返回原始 SQL，提示"部分语句无法格式化" |
